# 🗺️ Транспортный справочник с визуализацией маршрутов

Приложение для хранения, обработки и визуализации транспортных маршрутов (автобусы, метро и т.д.) с поддержкой JSON-ввода и графического интерфейса на Qt.

---

## 🚀 Возможности

- 📥 Загрузка данных из JSON-файлов
- ➕ Дозапись данных в каталог (без перезаписи)
- 🧹 Очистка базы данных
- 🔍 Запрос информации:
  - об остановке
  - о маршруте
  - построение карты

- 🗺️ Генерация SVG-карты маршрутов
- 🖥️ Удобный GUI на Qt (без консоли)

---

## 🧱 Архитектура проекта

Проект разделён на несколько независимых компонентов:

- **TransportCatalogue** — хранение остановок и маршрутов
- **JsonReader** — загрузка данных из JSON
- **RequestHandler** — обработка запросов
- **MapRenderer** — генерация SVG-карты
- **Qt GUI** — пользовательский интерфейс

---

## 📂 Формат входного JSON

Пример:

```json
{
  "base_requests": [
    {
      "type": "Stop",
      "name": "A",
      "latitude": 55.0,
      "longitude": 37.0,
      "road_distances": { "B": 1200 }
    },
    {
      "type": "Bus",
      "name": "256",
      "stops": ["A", "B"],
      "is_roundtrip": false
    }
  ],
  "render_settings": {
    "width": 800,
    "height": 600,
    "padding": 50,
    "stop_radius": 5,
    "line_width": 12,
    "color_palette": ["green", "red", "blue"]
  }
}
```

---

## 📊 Запросы

В GUI пользователь выбирает тип запроса:

### 🚌 Маршрут (Bus)

Выход:

```json
{
  "curvature": 1.2,
  "route_length": 4371,
  "stop_count": 12,
  "unique_stop_count": 7
}
```

---

### 🚏 Остановка (Stop)

```json
{
  "buses": ["256", "750"]
}
```

или

```json
{
  "error_message": "not found"
}
```

---

### 🗺️ Карта (Map)

- Результат сохраняется как `.svg`
- Используются параметры из `render_settings`

---

## 🖥️ Интерфейс

В приложении доступны:

- выбор файла с данными
- выбор типа запроса
- ввод имени остановки/маршрута
- задание имени выходного файла
- кнопки:
  - **Выполнить запрос**
  - **Очистить базу**

---

## ⚙️ Особенности реализации

- Данные хранятся в `deque` для стабильности указателей
- Используется `string_view` для оптимизации
- Рендеринг карты через собственную SVG-библиотеку
- Поддержка цветовой палитры (RGB и именованные цвета)

---

## ⚠️ Важно

- Повторная загрузка одного и того же файла **добавляет данные**, а не заменяет их
- При необходимости используйте кнопку **"Очистить базу"**
- Названия остановок должны быть уникальными

---

## 📌 Рекомендации по render_settings

Оптимальные диапазоны:

| Параметр       | Рекомендуемое значение |
| -------------- | ---------------------- |
| width / height | 400–2000               |
| padding        | 20–80                  |
| stop_radius    | 2–6                    |
| line_width     | 4–12                   |
